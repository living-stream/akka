# 内容生成 (Content Creation)

## 目标
从选题到成稿的全流程内容生产，产出高质量的小红书图文笔记。

## 核心原则
**禁止瞎编！** 涉及真实地点/店铺/产品/数据时，必须先用浏览器搜索验证，确保内容真实可信。

## 重要说明
- **此skill仅负责内容生成**，包括信息收集、文案撰写、图片生成
- **真实性偏好**，生成探店类真实内容前，必须先通过浏览器工具/对应技能完成真实调研，获取实际门店数据、价格、风味、评分等真实信息，禁止无依据虚构内容。
- **不负责内容发布**，如需发布请使用 `content-publishing skill`
- 如果用户说"生成并发布"，应先调用此skill生成内容，再调用 `content-publishing skill` 发布

## 判断是否需要信息收集

### 需要先搜索的情况：
- 涉及具体地点（如：上海咖啡店推荐）
- 涉及具体店铺/品牌（如：XX品牌测评）
- 涉及具体数据（如：2024年咖啡市场规模）
- 涉及具体产品（如：XX型号咖啡机评测）
- 涉及真实人物/事件

### 可以直接生成的情况：
- 纯观点/感悟类（如：独居生活的10个真相）
- 通用方法论（如：如何拍出好看的咖啡照）
- 个人经验分享（如：我的晨间routine）
- 情感共鸣类（如：打工人的周一心情）

## 工作流

### 1) 信息收集（如需要）
当内容涉及真实信息时：

1. **确定搜索平台**：
   - 地点/店铺 → 大众点评、小红书
   - 产品/品牌 → 淘宝、小红书、品牌官网
   - 数据/资讯 → 百度、知乎、行业报告

2. **执行搜索**：使用 `use_browser` 执行以下步骤：

**浏览器操作路径（探店场景）**：
```
Step 1: 打开大众点评，在搜索框输入"[地点] [店铺类型]"，点击搜索
Step 2: 浏览搜索结果列表，查看评分和评价数
Step 3: 点击第一家店铺进入详情页
Step 4: 记录店铺名称、地址、营业时间、评分、人均消费
Step 5: 浏览推荐菜/特色，记录 2-3 个招牌
Step 6: 浏览用户评价，记录 3-5 条好评亮点
Step 7: 返回搜索结果，重复 Step 3-6 收集 3-5 家店铺
```

**浏览器操作路径（产品测评场景）**：
```
Step 1: 打开小红书，搜索"[品牌名] [产品名]"
Step 2: 浏览搜索结果，查看相关笔记
Step 3: 点击高赞笔记，记录用户评价要点
Step 4: 打开淘宝，搜索同一产品
Step 5: 记录产品价格、规格、销量
Step 6: 浏览商品评价，记录用户反馈
```

**use_browser 调用示例**：
```
Step 1: 打开大众点评，搜索"安福路 咖啡店"
Step 2: 点击第一家店，记录店名、地址、评分
Step 3: 记录招牌产品和用户评价亮点
Step 4: 返回搜索结果，收集3家店的信息
```

3. **提取关键信息**：
   - 店铺名称、地址、营业时间
   - 特色产品/服务
   - 用户评价亮点
   - 图片参考

4. **整理成素材**：
   - 将收集到的真实信息整理成结构化素材
   - 作为后续内容生成的输入

### 2) 内容生成
1. **调用生成工具**：
   ```
   generate_note(
       platform="小红书",
       instruction="[包含真实素材的创作指令]",
       output_path="workspace/notes/[笔记主题]"
   )
   ```

2. **指令构建技巧**：
   - 明确选题角度（如：从"打工人续命"角度写咖啡店推荐）
   - 提供真实素材（如：以下是从大众点评收集的5家店...）
   - 指定风格要求（如：用技术隐喻、保持沪上精致感）

3. **输出内容**：
   所有文件将保存在 `workspace/notes/[笔记主题]/` 目录下：
   - `outline.md`：内容大纲
   - `copywriting.md`：标题、正文
   - `images/`：配图

### 3) 质量检查
生成后检查：
- [ ] 标题是否有吸引力（数字/痛点/悬念）
- [ ] 正文是否通顺、有结构
- [ ] 是否包含真实信息（如涉及）
- [ ] 图片数量是否足够（建议 3-6 张）

## 示例场景

### 场景1：探店笔记（需要搜索）
**用户指令**："帮我写一篇上海安福路精品咖啡探店笔记"

**执行流程**：
1. 先用 `use_browser` 在大众点评搜索"安福路精品咖啡"
2. 收集 3-5 家店的：店名、地址、特色、评价
3. 调用 `generate_note`，指令中包含收集到的真实信息
4. 质量检查：标题、正文、图片
5. 返回生成结果

### 场景2：观点笔记（无需搜索）
**用户指令**："写一篇关于独居生活幸福感的笔记"

**执行流程**：
1. 直接调用 `generate_note`
2. 基于人设和经验生成内容
3. 质量检查：标题、正文、图片
4. 返回生成结果

### 场景3：产品测评（需要搜索）
**用户指令**："测评一下三顿半咖啡"

**执行流程**：
1. 用 `use_browser` 在小红书/淘宝搜索"三顿半咖啡"
2. 收集：产品线、价格、口味、用户评价
3. 调用 `generate_note` 生成测评内容
4. 质量检查：标题、正文、图片
5. 返回生成结果

## 常见问题处理
1. **搜索不到足够信息**：扩大搜索范围或换平台
2. **信息有冲突**：以官方信息为主，用户评价为辅
3. **生成内容太泛**：在指令中强调"具体、真实、有细节"

## 完成标准
**内容质量**：
- 内容真实可信（涉及真实信息时已验证）
- 标题有吸引力
- 正文结构清晰、有干货
- 图片与内容匹配
- 输出文件完整（workspace/notes/[笔记主题]/ 目录下包含 outline.md, copywriting.md, images/）

**注意**：此skill仅负责内容生成，不负责发布。如需发布，请使用 content-publishing skill。
