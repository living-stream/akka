# Task Planning (宏观任务规划)

## 目标
将用户的模糊意图或宏大目标，转化为结构清晰、可执行的 Task 列表。

## 核心定位
- **Task Planning**: 解决 **"做什么" (What)** 的问题。针对宏大任务或目标的拆解。
- **Action Planning**: 解决 **"怎么做" (How)** 的问题。针对具体 Task 的执行路径拆解（请使用 `action-planning` skill）。

## 核心概念：三级颗粒度

### 1. Objective (O)
- **定义**：定性的、鼓舞人心的最终目标。
- **适用**：长期战略规划。
- **示例**：“打造小红书 AI 领域的头部资讯账号。”

### 2. Key Results (KR)
- **定义**：定量的、可衡量的阶段性成果。必须包含**可行性验证**。
- **适用**：验证核心机制是否跑通，或确认阶段性里程碑。
- **示例**：“建立每周选题-发文-复盘的机制，并且确保这个机制可以持续涨粉，验证可行性。”

### 3. Task (任务)
- **定义**：为达成 KR 需要完成的具体工作事项。相当于领导给实习生布置的一个完整任务。
- **粒度**：明确“做什么”，通常是一个独立的交付单元。
- **示例**：
  - “选取 3 个对标账号，分析其爆款逻辑。”
  - “搜索‘AI工具’关键词，找到其中最热门的 5 篇内容。”

## 总体策略
1. **区分复杂度**：
   - **长期/复杂**：使用 O -> KR -> Task 的完整拆解。
   - **短期/简单**：直接产出 Task List。
2. **可行性验证 (Critical)**：对于具有不确定性的任务（如涨粉），KR 必须先包含“验证期（MVP）”，再进入“增长期”。

## 工作流

### 模式 A：长期复杂规划 (O-KR-Task)
1. **设定 O**：明确最终愿景。
2. **拆解 KR**：
   - **KR1 (验证期)**：建立机制，验证 MVP 有效性（如：确保SOP能带来稳定流量）。
   - **KR2 (增长期)**：在机制验证通过后，设定规模化增长指标。
3. **拆解 Task**：将每个 KR 拆解为具体的 Task 列表。

### 模式 B：短期简单任务 (Task List)
1. **明确需求**：确认用户需要完成的具体事项。
2. **产出 Task List**：列出需要执行的 Task。

## 示例：找对标账号

**Task List**:
1. Task: 确认用户想做的账号类型（如 AI 资讯）。
2. Task: 使用关键词“AI 资讯”在小红书搜索高赞图文笔记。
3. Task: 分析高赞笔记作者主页，筛选出垂直度高的对标账号。
4. Task: 整理并汇报对标账号信息。

## 输出规范 (Required Output)

每次执行本 Skill，**必须** 输出以下两个独立部分：

### 1. 战略拆解视图 (Strategic Logic)
*目标：清晰展示拆解思路和逻辑，解释“为什么这么拆”。*
- **Objective**: ...
- **Key Result 1**: ... (Metric: ...)
  - **Task 1.1**: ... (Rationale: ...)
  - **Task 1.2**: ...
- **Key Result 2**: ...

### 2. 执行排期视图 (Execution Schedule)
*目标：明确执行顺序、依赖关系和时间成本。*
- **时间粒度**：按实际情况评估为 **小时 (h)** 或 **天 (d)**。
- **格式示例**：
  | ID | Task Name | 依赖前置 (Dependencies) | 预计耗时 (Est. Time) |
  |:--:|:----------|:-----------------------:|:--------------------:|
  | T1 | 调研竞品   | -                       | 2h                   |
  | T2 | 撰写初稿   | T1                      | 0.5d                 |
  | T3 | ...       | ...                     | ...                  |

## 下一步建议
完成 Task Planning 后，针对其中的每个具体 Task，**强烈建议**调用 `action-planning` skill 进行详细的执行路径拆解，以确保执行的准确性和鲁棒性。
