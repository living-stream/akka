# 编辑人设 (Edit Soul)

根据用户指令更新 soul.md 文件。

## 什么是 soul.md？

soul.md 定义了用户的人设、身份、语气和行为准则。它决定了 agent 如何沟通和创作内容。文件位置：`users/{uid}/soul.md`

## 何时使用此 Skill

在以下情况下触发此 skill：
- 用户想更新人设/身份
- 用户想改变语气/风格
- 用户想修改生活方式/职业细节
- 用户说"更新我的档案"、"修改人设"、"编辑 soul.md"
- 用户对内容风格提出反馈，需要反映到人设中

## 工作流程

### 1. 理解请求

首先，理解用户想修改 soul 的哪个方面：
- **身份**：姓名、地点、职业、生活方式
- **理念**：核心价值观、信念、原则
- **语气风格**：沟通风格、语言模式
- **内容策略**：写作风格、互动模式
- **系统指令**：行为准则

### 2. 读取当前 soul.md

读取用户当前的 soul.md：
```
Read users/{uid}/soul.md
```

### 3. 澄清修改内容（如需要）

如果请求模糊，询问澄清问题：
- "您想更新哪个部分？"
- "具体想做什么修改？"
- "是添加新内容还是修改现有内容？"

### 4. 应用修改

根据用户指令更新 soul.md 文件：
- 保留结构和格式
- 只修改请求的部分
- 与其他部分保持一致性
- 保持语气自然真实

### 5. 确认修改

更新后，向用户展示：
- 修改了什么
- 更新后的内容
- 询问是否需要进一步调整

## 示例场景

### 场景 1：更新职业
**用户**："我换工作了，现在在字节跳动做算法工程师"

**操作**：
1. 读取当前 soul.md
2. 更新"职业"部分
3. 更新相关部分（如生活方式、语气）
4. 展示修改内容

### 场景 2：改变语气
**用户**："我想让语气更活泼一点，多用emoji"

**操作**：
1. 读取当前 soul.md
2. 更新"语气风格"部分
3. 添加 emoji 使用指南
4. 展示修改内容

### 场景 3：添加新部分
**用户**："我想加一个关于咖啡偏好的部分"

**操作**：
1. 读取当前 soul.md
2. 添加新部分（如"## 6. 咖啡偏好"）
3. 如需要，询问具体偏好
4. 展示修改内容

## 重要准则

1. **保留身份**：除非明确要求，否则不要完全重写人设
2. **保持一致性**：确保修改不与其他部分冲突
3. **自然真实**：人设应该感觉真实，不造作
4. **重大修改前确认**：在进行重大修改前与用户确认
5. **备份原始内容**：考虑在修改前展示原始内容

## 文件路径解析

Skill 应使用用户上下文确定正确的文件路径：
- 从上下文获取当前用户 ID
- 读取并更新 `users/{uid}/soul.md`

## 错误处理

- 如果 soul.md 不存在，提示用户先创建
- 如果用户 ID 不可用，请求澄清
- 如果请求的修改不清晰，询问更多细节
- 如果修改会导致人设不一致，警告用户

## 成功标准

soul.md 更新成功的标准：
- 文件正确更新
- 修改反映了用户意图
- 保留结构和格式
- 用户确认修改正确
