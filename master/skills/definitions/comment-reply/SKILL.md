# 评论回复 (Comment Reply)

## 目标
自动回复小红书上的评论，增加粉丝互动，提升账号活跃度。

## 核心原则
- **人设一致**：回复风格要符合用户人设
- **有趣优先**：挑选有回复价值的评论，避免无脑回复
- **真诚互动**：不敷衍，每条回复都要有诚意
- **适度回复**：一次回复不超过 5 条，避免刷屏

## 工作流

### 1) 获取用户人设
使用 `use_browser` 调用 `get_user_persona` action 获取用户人设：

```
Step 1: 调用 get_user_persona 获取用户人设内容
Step 2: 记住人设风格，用于后续生成回复
```

### 2) 访问通知页面
使用 `use_browser` 执行以下步骤：

```
Step 1: 打开 https://www.xiaohongshu.com/notification
Step 2: 点击"评论和@"标签，切换到评论通知列表
Step 3: 浏览评论列表，查看评论内容
```

### 3) 筛选评论
从评论列表中挑选 <= 5 条值得回复的评论：

**优先回复**：
- 有趣/有深度的评论
- 提问类评论
- 情感共鸣类评论
- 点赞数较高的评论

**跳过的评论**：
- 纯表情/无意义评论（如："👍""支持"）
- 广告/推销评论
- 已经回复过的评论
- 恶意攻击/引战评论

### 4) 生成回复
根据用户人设生成回复：
- 回复风格要符合人设特点
- 回复要有个性，避免客服腔
- 保持真诚，不敷衍

### 5) 发布回复
使用浏览器工具发布回复：

```
Step 1: 点击目标评论的回复按钮
Step 2: 在输入框中输入回复内容
Step 3: 点击发送按钮
Step 4: 确认回复成功
```

### 6) 记录互动
回复完成后：
- 如果有有趣的或者深刻的，使用 `update_memory` 记录有趣的互动
- 记录粉丝的常见问题类型

## 注意事项
1. **不要过度回复**：一次回复 <= 5 条，避免被判定为刷屏
2. **保持人设**：回复要符合用户人设，不要变成客服腔
3. **避免争议**：不回复引战评论，不参与骂战
4. **记录学习**：有趣的互动可以记录到 memory

## 完成标准
- 成功获取用户人设
- 成功访问通知页面并筛选评论
- 回复内容符合用户人设风格
- 回复已成功发布
- 有趣互动已记录到 memory
